01
PROJECT SETUP
~30 min · Get Unity 6 ready · Create your 2D project
✓ DONE
▶
✓
Open Unity Hub and create a new project SETUP
Open Unity Hub → click New Project → select the 2D Core template → name it SpaceInvaders → click Create. Unity 6 will set up the project. This takes a minute or two — normal.
Choose 2D Core, not 2D URP. Simpler for a first project and everything in this plan is written for it.
✓
Familiarise yourself with the layout SETUP
Take 5 minutes to look around. The Scene view is your visual workspace. The Game view is what the player sees. The Hierarchy (top left) lists every object in the scene. The Inspector (right) shows the selected object's settings. The Project panel (bottom) is your file system.
Don't touch anything yet — just look. Knowing the layout saves huge amounts of time later.
✓
Set the camera and scene boundaries SCENE
Click Main Camera in the Hierarchy. In the Inspector, set Size to 5. This controls how much of the world we see. In the Inspector, also set Background Color to black (R:0 G:0 B:0). Create an empty GameObject (right-click Hierarchy → Create Empty) and name it GameManager — you'll use this later.
✓
Create your folder structure SETUP
In the Project panel, right-click the Assets folder → Create → Folder. Create these four folders: Scripts, Sprites, Prefabs, Audio. Organised projects are faster to work in — do this now rather than later.
02
PLAYER SHIP
~60 min · Create the ship · Make it move · Keep it inside the screen
✓ DONE
▶
✓
Create the player sprite SPRITE
Right-click in the Hierarchy → 2D Object → Sprites → Triangle. Rename it Player in the Hierarchy. In the Inspector, set its color to green. Set its Transform Position to (0, -4, 0) — this puts it near the bottom. Scale it to (0.8, 0.8, 1). This is your ship for now — you can replace it with a proper sprite later without breaking anything.
Using Unity's built-in shapes first means you can build and test all the logic before worrying about art.
✓
Add a Rigidbody2D and Collider to the player SCENE
With Player selected, click Add Component in the Inspector. Add Rigidbody2D. Then in the Rigidbody2D settings, set Gravity Scale to 0 (space has no gravity) and set Body Type to Kinematic. Then add a PolygonCollider2D. This gives your ship a physical presence in the game world.
✓
Write the PlayerController script C# CODE
In the Project panel, right-click Scripts folder → Create → C# Script. Name it PlayerController. Double-click to open in Visual Studio. Replace everything with:
using UnityEngine; public class PlayerController : MonoBehaviour { public float speed = 8f; private float screenBound = 8.5f; void Update() { // Get left/right input (arrow keys or A/D) float move = Input.GetAxis("Horizontal"); transform.Translate(Vector2.right * move * speed * Time.deltaTime); // Clamp so the ship can't leave the screen float clampedX = Mathf.Clamp(transform.position.x, -screenBound, screenBound); transform.position = new Vector3(clampedX, transform.position.y, 0); } }
Save the file (Ctrl+S), go back to Unity, drag the PlayerController script onto the Player object in the Hierarchy. Press Play and test — the ship should move left and right.
03
BULLETS & SHOOTING
~45 min · Create a bullet · Fire on spacebar · Auto-destroy off screen
✓ DONE
▶
✓
Create the bullet object and prefab SCENE
Right-click Hierarchy → 2D Object → Sprites → Square. Name it Bullet. Scale it to (0.1, 0.4, 1) — tall and thin. Color it yellow. Add a Rigidbody2D (Gravity Scale: 0, Body Type: Dynamic) and a BoxCollider2D. Then drag the Bullet from the Hierarchy down into your Prefabs folder in the Project panel. This creates a reusable prefab. Delete the Bullet from the Hierarchy — the prefab is what matters now.
✓
Write the Bullet script C# CODE
Create a new C# Script in the Scripts folder called Bullet. Add this code:
using UnityEngine; public class Bullet : MonoBehaviour { public float speed = 12f; void Update() { transform.Translate(Vector2.up * speed * Time.deltaTime); // Destroy bullet when it leaves the top of screen if (transform.position.y > 6f) Destroy(gameObject); } }
Drag this Bullet script onto the Bullet prefab in the Prefabs folder (not the scene — the prefab itself).
✓
Add shooting to PlayerController C# CODE
Open PlayerController.cs again. Add shooting logic so the player fires on spacebar:
using UnityEngine; public class PlayerController : MonoBehaviour { public float speed = 8f; public GameObject bulletPrefab; public float fireRate = 0.4f; private float nextFire = 0f; private float screenBound = 8.5f; void Update() { float move = Input.GetAxis("Horizontal"); transform.Translate(Vector2.right * move * speed * Time.deltaTime); float clampedX = Mathf.Clamp(transform.position.x, -screenBound, screenBound); transform.position = new Vector3(clampedX, transform.position.y, 0); // Fire on spacebar, limited by fire rate if (Input.GetKey(KeyCode.Space) && Time.time > nextFire) { nextFire = Time.time + fireRate; Instantiate(bulletPrefab, transform.position, Quaternion.identity); } } }
Back in Unity, select the Player object. You'll see a "Bullet Prefab" slot in the Inspector. Drag your Bullet prefab from the Prefabs folder into that slot. Press Play and test shooting.
04
THE INVADERS
~75 min · Create one invader · Form a grid · Make them march and drop
✓ DONE
▶
✓
Create the Invader prefab SCENE
Right-click Hierarchy → 2D Object → Sprites → Square. Name it Invader. Scale to (0.7, 0.7, 1). Color it red. Add a BoxCollider2D. Drag it into your Prefabs folder. Delete it from the Hierarchy. You'll spawn these programmatically.
✓
Write the InvaderGrid script C# CODE
Create a new C# Script called InvaderGrid. This script spawns and moves all the invaders:
using UnityEngine; using System.Collections; public class InvaderGrid : MonoBehaviour { public GameObject invaderPrefab; public int rows = 4, cols = 9; public float xSpacing = 1.2f, ySpacing = 1f; public float moveSpeed = 1.5f; public float dropAmount = 0.4f; private float direction = 1f; private float edgeLimit = 7f; private bool needsDrop = false; void Start() { // Spawn the grid of invaders for (int r = 0; r < rows; r++) for (int c = 0; c < cols; c++) { Vector3 pos = transform.position + new Vector3(c * xSpacing, -r * ySpacing, 0); Instantiate(invaderPrefab, pos, Quaternion.identity, transform); } } void Update() { transform.Translate(Vector2.right * direction * moveSpeed * Time.deltaTime); // Check if any invader has hit the edge foreach (Transform child in transform) { if (child.position.x > edgeLimit || child.position.x < -edgeLimit) { direction *= -1f; transform.Translate(Vector2.down * dropAmount); break; } } // Speed up as invaders are destroyed int remaining = transform.childCount; moveSpeed = Mathf.Lerp(4f, 1.5f, remaining / (float)(rows * cols)); } }
Create an empty GameObject in the Hierarchy named InvaderGrid. Set its position to (-4.8, 2, 0). Attach the InvaderGrid script to it. Drag the Invader prefab into the Invader Prefab slot. Press Play — you should see a grid of red squares marching back and forth.
✓
Add enemy bullets (invaders shoot back) C# CODE
Create a new script called EnemyShooter and attach it to the Invader prefab:
using UnityEngine; public class EnemyShooter : MonoBehaviour { public GameObject bulletPrefab; private float nextShot; void Start() { // Random initial delay so not all invaders fire at once nextShot = Time.time + Random.Range(2f, 8f); } void Update() { if (Time.time > nextShot) { Instantiate(bulletPrefab, transform.position, Quaternion.identity); nextShot = Time.time + Random.Range(3f, 9f); } } }
Create a second bullet prefab called EnemyBullet — make it red/orange and moving downward (change Vector2.up to Vector2.down in the Bullet script, or make a separate EnemyBullet script). Assign it to the EnemyShooter on the Invader prefab.
05
COLLISION & GAME LOGIC
~60 min · Bullets destroy invaders · Enemy bullets kill the player · Win and lose conditions
✓ DONE
▶
✓
Set up Physics Layers SCENE
Go to Edit → Project Settings → Physics 2D → Layer Collision Matrix. First, create layers: go to the top right of Unity → Layers → Edit Layers. Add: Player (Layer 6), Invader (Layer 7), PlayerBullet (Layer 8), EnemyBullet (Layer 9). Then assign each object its layer via the Inspector. In the collision matrix, uncheck PlayerBullet vs PlayerBullet, EnemyBullet vs EnemyBullet, Player vs PlayerBullet, Invader vs EnemyBullet.
Layers prevent bullets from colliding with things they shouldn't. This stops your own bullets from hitting your ship.
✓
Add collision detection to Bullet C# CODE
Update Bullet.cs to handle hitting an invader:
void OnTriggerEnter2D(Collider2D other) { if (other.CompareTag("Invader")) { GameManager.instance.AddScore(10); Destroy(other.gameObject); Destroy(gameObject); } }
Add the Tag "Invader" to the Invader prefab (Inspector → Tag dropdown → Add Tag). Make both colliders Is Trigger = true on the bullets.
✓
Write the GameManager script C# CODE
Create GameManager.cs and attach it to the GameManager object. This is the brain of the game:
using UnityEngine; using UnityEngine.SceneManagement; public class GameManager : MonoBehaviour { public static GameManager instance; public int score = 0; public int lives = 3; void Awake() { // Singleton — only one GameManager exists if (instance == null) instance = this; else Destroy(gameObject); } public void AddScore(int points) { score += points; UIManager.instance.UpdateScore(score); } public void PlayerHit() { lives--; UIManager.instance.UpdateLives(lives); if (lives <= 0) GameOver(); } public void GameOver() { UIManager.instance.ShowGameOver(); Time.timeScale = 0f; // Pause the game } public void RestartGame() { Time.timeScale = 1f; SceneManager.LoadScene(SceneManager.GetActiveScene().name); } }
✓
Handle player getting hit C# CODE
Add collision detection to PlayerController.cs:
void OnTriggerEnter2D(Collider2D other) { if (other.CompareTag("EnemyBullet")) { Destroy(other.gameObject); GameManager.instance.PlayerHit(); } }
Add the tag "EnemyBullet" to your enemy bullet prefab the same way you added "Invader".
06
UI — SCORE · LIVES · SCREENS
~60 min · Start screen · HUD · Game Over screen
✓ DONE
▶
✓
Create the Canvas and HUD SCENE
Right-click Hierarchy → UI → Canvas. Unity creates a Canvas and EventSystem automatically. On the Canvas, set Render Mode to Screen Space - Overlay. Inside the Canvas, right-click → UI → Text - TextMeshPro (click Import when prompted). Name it ScoreText. Set its text to "SCORE: 0". Anchor it to the top-left. Create another Text called LivesText, anchored top-right, text "LIVES: 3".
✓
Create Game Over and Start screen panels SCENE
Inside the Canvas, right-click → UI → Panel. Name it GameOverPanel. Set its color to black with some transparency. Add a TextMeshPro text inside it saying "GAME OVER". Add a UI → Button inside it, label it "RESTART". Duplicate the panel, name it StartPanel, change text to "SPACE INVADERS — Press any key to start". Set GameOverPanel and StartPanel to inactive by unchecking the checkbox next to their name in the Inspector.
✓
Write the UIManager script C# CODE
Create UIManager.cs and attach it to the Canvas:
using UnityEngine; using TMPro; public class UIManager : MonoBehaviour { public static UIManager instance; public TMP_Text scoreText, livesText; public GameObject gameOverPanel, startPanel; void Awake() { instance = this; } void Start() { startPanel.SetActive(true); Time.timeScale = 0f; } void Update() { if (startPanel.activeSelf && Input.anyKeyDown) { startPanel.SetActive(false); Time.timeScale = 1f; } } public void UpdateScore(int s) { scoreText.text = "SCORE: " + s; } public void UpdateLives(int l) { livesText.text = "LIVES: " + l; } public void ShowGameOver() { gameOverPanel.SetActive(true); } }
Wire up the Restart button: select the Button in the Inspector, click + under OnClick(), drag the GameManager object into the slot, and select GameManager → RestartGame from the dropdown.
07
SOUND EFFECTS
~30 min · Shoot sound · Explosion · Game over sting
✓ DONE
▶
✓
Get free sound effects AUDIO
Go to freesound.org (free account required) and download three short sounds: a laser/shoot sound, an explosion sound, and a game-over jingle. Alternatively, use sfxr.me in the browser — it generates retro game sounds with one click. Export as .wav. Drag all sounds into your Audio folder in Unity.
sfxr.me is the fastest option — it has laser, explosion, and lose presets built in. Takes 2 minutes total.
✓
Add AudioSource and wire up sounds C# CODE
Add an AudioSource component to the GameManager object (Add Component → Audio → AudioSource, uncheck Play On Awake). Then update GameManager.cs to include sound playback:
public AudioClip shootSound, explosionSound, gameOverSound; private AudioSource audioSource; void Awake() { if (instance == null) instance = this; else Destroy(gameObject); audioSource = GetComponent<AudioSource>(); } public void PlayShoot() { audioSource.PlayOneShot(shootSound); } public void PlayExplosion() { audioSource.PlayOneShot(explosionSound); } public void PlayGameOver() { audioSource.PlayOneShot(gameOverSound); }
Call GameManager.instance.PlayShoot() in PlayerController when firing, PlayExplosion() in Bullet when it hits an invader, and PlayGameOver() in the GameOver() method. Drag your audio clips into the Inspector slots.
08
POLISH & BUILD
~30 min · Test everything · Final tweaks · Export a playable build
✓ DONE
▶
✓
Full playthrough test TEST
Press Play and run through the complete game: start screen appears → press key to start → ship moves and fires → invaders march and shoot back → killing all invaders should be tested → taking hits reduces lives → third hit triggers game over screen → restart button works. Fix any bugs you find. Common issues: bullets passing through things (check Is Trigger and layers), invaders not dying (check tags), UI not updating (check UIManager references).
✓
Tune the game feel POLISH
Spend 10 minutes adjusting values in the Inspector while playing: player speed (try 6–10), bullet speed (try 10–15), fire rate (try 0.3–0.5), invader speed, grid size. These are all public variables so you can tweak them live in Play mode. Small changes make a big difference to how fun it feels. Write down your final values before stopping Play mode — changes made in Play mode are lost.
Unity discards Inspector changes made during Play mode when you stop. Always note down your tuned values and re-enter them after stopping.
✓
Build and export the game BUILD
Go to File → Build Settings. Make sure your scene is listed (if not, click Add Open Scenes). Platform: Windows / Mac / Linux. Click Build. Choose a folder. Unity will compile and produce a playable .exe (Windows) or .app (Mac). That's a real game you built in a day.
WebGL build lets you share it in a browser. It takes longer to build but means anyone can play it via a link — worth doing if you want to share it easily.
